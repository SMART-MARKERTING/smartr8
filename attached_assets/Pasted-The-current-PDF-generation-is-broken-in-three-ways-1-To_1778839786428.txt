The current PDF generation is broken in three ways:
1. Top header is cut off horizontally
2. Content duplicates across page breaks (same paragraph 
   appears at end of page 1 AND start of page 2)
3. Excessive blank pages at the bottom

Root cause: html2canvas + jsPDF is capturing the full DOM as 
a single tall image, then slicing with overlap and including 
empty whitespace in the canvas height. CSS page-break rules 
don't help because the slicing is image-based, not DOM-based.

============================================
SOLUTION: SWITCH TO SERVER-SIDE PDF VIA PUPPETEER
============================================

Replace the current html2canvas + jsPDF implementation entirely 
with Cloudflare Browser Rendering (Puppeteer-based). This 
respects real CSS page rules and produces clean PDFs without 
slicing artifacts.

STEP 1: Install Cloudflare Puppeteer

npm install @cloudflare/puppeteer

STEP 2: Enable Browser Rendering in Cloudflare
Tell the user: "Go to Cloudflare dashboard → your account → 
Workers & Pages → your smartr8 project → Settings → Functions 
→ enable Browser Rendering binding."
Free tier covers 10 minutes/day which is plenty.

STEP 3: Replace /api/worksheet/generate-pdf endpoint

The new endpoint should:
1. Accept worksheet inputs as JSON POST body
2. Use Puppeteer to launch a headless browser
3. Navigate to a server-rendered version of the worksheet 
   at a hidden route (e.g., /worksheet/render?data=<base64>) 
   that injects the inputs and renders the full HTML
4. Wait for the page to fully load (networkidle)
5. Generate PDF with these options:

   await page.pdf({
     format: 'letter',
     printBackground: true,
     margin: {
       top: '0.4in',
       right: '0.4in',
       bottom: '0.4in',
       left: '0.4in'
     },
     preferCSSPageSize: true
   });

6. Return the PDF buffer as the response

STEP 4: Add print-aware CSS to the worksheet template

In the worksheet HTML/CSS, add:

@media print {
  @page {
    size: letter;
    margin: 0.4in;
  }
  
  body {
    margin: 0;
    padding: 0;
    -webkit-print-color-adjust: exact;
    print-color-adjust: exact;
  }
  
  /* Hide the input panel completely in print */
  .input-panel {
    display: none !important;
  }
  
  /* Show only the worksheet itself */
  .worksheet {
    box-shadow: none;
    margin: 0;
    max-width: 100%;
  }
  
  /* Keep these elements together — never split */
  table,
  .banner,
  .signature-block,
  .compliance-footer,
  .header-bar {
    page-break-inside: avoid;
    break-inside: avoid;
  }
  
  /* Headers stay with their following content */
  h1, h2, h3 {
    page-break-after: avoid;
    break-after: avoid;
  }
  
  /* Each major section can break before if needed */
  .signature-block {
    page-break-before: auto;
    break-before: auto;
  }
}

STEP 5: Fix the top header cropping

The current cropping suggests negative margin or overflow 
issue in the .header-bar element. Audit the worksheet 
template:

- Remove any negative margins on .header-bar that pull it 
  above the printable area
- Ensure .header-bar uses padding (not margin) for spacing
- Confirm body has zero margin/padding when @media print is 
  active

STEP 6: Fix the duplicate content bug

This goes away automatically once Puppeteer handles page 
breaks via CSS. No more image slicing means no more overlap 
artifacts.

STEP 7: Fix the trailing blank space

The canvas was capturing the full body height including the 
input panel and any scroll padding. Server-side rendering 
only captures the .worksheet div content, eliminating this 
entirely.

In the render route, wrap output to only include the 
worksheet HTML, not the input panel or modal markup.

============================================
FALLBACK IF CLOUDFLARE BROWSER RENDERING ISN'T AVAILABLE
============================================

If Browser Rendering is not enabled or can't be enabled, use 
@react-pdf/renderer as a fallback:

npm install @react-pdf/renderer

Rebuild the worksheet output as a React PDF component 
(separate from the screen view). Use Document, Page, View, 
Text components with explicit page-break logic. This is more 
work but produces pixel-perfect PDFs without browser 
dependencies.

Do NOT try to fix the existing html2canvas + jsPDF approach. 
That path has fundamental limitations and the user has 
already hit them.
